## Course Description

**Object Oriented Design (OOD)** focuses on the principles and practices of designing robust, reusable, and maintainable software systems using object-oriented concepts. The course builds on programming fundamentals and introduces students to software modeling, design principles, and architectural thinking.

Students learn how to structure complex systems using objects, classes, relationships, and design patterns to produce clean and scalable software solutions.

---

## Topics Covered

- Introduction to Software Design Principles
- Object-Oriented Analysis and Design (OOAD)
- Classes, Objects, and Relationships
- Encapsulation, Inheritance, and Polymorphism
- SOLID Principles of Design
- UML Diagrams (Class, Sequence, Use Case)
- Design Patterns (Creational, Structural, Behavioral)
- Software Architecture Basics
- Refactoring and Code Optimization
- Modularity and Reusability
- Agile Design Practices
- Case Studies in Software Design

---

## Learning Outcomes

By the end of the course, students will be able to:

- Apply object-oriented principles in software design.
- Model systems using UML diagrams.
- Design scalable and maintainable software architectures.
- Understand and apply common design patterns.
- Analyze software requirements and translate them into object models.
- Improve code quality through refactoring techniques.

---

## Teaching Methodology

The course includes:

- Interactive lectures
- UML modeling exercises
- Case-study analysis
- Design pattern demonstrations
- Group design projects
- Peer code and design reviews

Students are encouraged to think beyond coding and focus on system-level software design.
